Contratos inteligentes

Na prática, um contrato inteligente ou smart contract é algo simples: um programa executando no ecossistema Ethereum. De forma geral, um contrato contém um conjunto de ações (código) e dados (estado) que residem em um endereço na Blockchain.

Para escrever um contrato, você precisa aprender a codificar em uma linguagem de programação apropriada, como Solidity utilizada pelo Ethereum. Nesta unidade, você conhecerá um pouco melhor sobre essa linguagem de programação.

IMPORTANTE Além do Solidity, existem outras linguagens de programação para desenvolvimento de contratos inteligentes. Outra linguagem amplamente utilizada e ativa é o Vyper.

Sabendo que os contratos inteligentes são um tipo de conta e que sua criação e execução são transações, talvez você esteja pensando: Vou precisar gastar Ether para testar meus contratos na rede com o pagamento do Gás (taxa)?

A resposta é não! É possível dispor de algumas ferramentas de teste que geram um ambiente simulado, onde é utilizada uma moeda sem valor real.

Antes de falar melhor a linguagem de programação Solidity, você entenderá como ter um ambiente de desenvolvimento para implementar seus primeiros contratos inteligentes, assim como simular o ecossistema para realização de testes.

Linguagem de Programação Solidity

O Solidity é uma das linguagens de programação de alto nível utilizada, principalmente, para a criação de contratos inteligentes da plataforma Ethereum,

VOCÊ SABIA? Uma linguagem de programação é considerada de alto nível quando possuem sintaxe mais próxima da linguagem humana do que da linguagem de máquina

É importante destacar seu uso para a criação de contratos, mas não para manutenção. Isso porque, os contratos não podem ser modificados após publicação, devido a propriedade de imutabilidade da blockchain.

Descrito em sua documentação oficial (https://docs.soliditylang.org/en/v0.8.17/), Solidity é uma linguagem orientada a objetos, influenciada pelo C++, Python e pelo JavaScript.

você sabia? A orientação a objetos é um paradigma da programação baseado em objetos definidos por classes. Os objetos podem possuir atributos e comportamentos.

É também uma linguagem tipada estaticamente, com suporte à criação de tipos definidos pelo usuário,

VOCÊ SABIA? Uma linguagem é considerada tipada estaticamente quando é necessário declarar explicitamente o tipo de cada dado utilizado, por exemplo variáveis, parâmetros de funções, valores de retorno, entre outros

O Solidity evolui muito rapidamente e você deve ficar atento às mudanças consultando a documentação constantemente. Além da melhoria da própria linguagem, essas mudanças acontecem também por causa da evolução do próprio Ethereum.

Agora que o escopo do contrato foi definido, você pode descrever as propriedades ( definição de variáveis e constantes), construtor e funções. Não existe um único formato correto, mas para facilitar a construção, considere a estrutura a seguir:

A declaração a seguir mostra uma variável chamada “mensagem”. Primeiro, foi definido seu tipo como string (texto) e logo após, tem-se o nome da variável (“mensagem”) A declaração é finalizada com ; (ponto e vírgula), assim como todos os comandos em Solidity.

É possível definir um valor inicial para a variável durante a declaração, mas optou-se por se fazer no construtor para que conheça um pouco sobre ele.

VOCÊ SABIA? O construtor funciona como uma função especial que é chamada sempre que uma nova instância do objeto contrato for criada. Nele podemos definir inicialização do valor das variáveis do objeto, entre outras.

A imagem a seguir mostra que o valor de uma variável do tipo string deve ficar entre aspas.

Agora que a variável foi declarada e inicializada, você pode programar a função que retorna seu valor.

Neste instante é importante que você entenda sobre a estrutura da função, não se preocupe em se aprofundar nos outras possibilidades oferecidas pelo Solidity.

Inicia-se a função com a palavra reservada function e coloca-se o nome da função e seus parâmetros entre parênteses, nesse caso o nome escolhido é “olaMundo()” e não há parâmetros, por isso os parênteses estão vazios.

Logo após, é definido o modificador de visibilidade de acesso. Neste caso, foi definido public, indicando que a função pode ser chamada de fora deste contrato.

Na sequência é definido o tipo da função, neste caso é um função do tipo view, pois é utilizada para visualizar uma informação.

Por fim, é definido o tipo de retorno usando a palavra reservada returns, neste caso o retorno é tipo string memory.

No escopo da função, a única instrução que ela executa é retornar o valor da variável “mensagem” utilizando a palavra reservada return. A imagem a seguir, mostra essa construção.
