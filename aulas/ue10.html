Sintaxe

Tipos de dados

Na linguagem de programação Solidity, o programador precisa especificar o tipo de cada variável que será usada, por isso ela é dita uma linguagem estaticamente tipada. Exemplos de outras linguagens tipadas incluem o Java, C, C++, dentre várias outras.

Dentre as principais vantagens de se trabalhar uma linguagem estaticamente tipada estão a facilidade de identificar erros no código e maior facilidade na leitura e entendimento do código em si.

O Solidity trabalho com três tipos principais de dados:

  1. Os Tipos de Valor (Value Types);
  2. Os Tipos de Referência (Reference Types);
  3. Os Tipos de Mapeamento (Mapping Types).

Tipos de valor

Os Value Types são utilizados para definir o tipo básicos de variáveis. Representam os tipos mais comuns, existentes em basicamente todas as linguagens de programação modernas. A seguir, será explorado alguns tipos e você entenderá como ele é representado em detalhes, e seu propósito de uso. Para ter acesso a detalhes de todos os tipos, acesse a documentação oficial do Solidity: https://docs.soliditylang.org/en/v0.8.17/types.html

Além disso, cada tipo admite um conjunto bem limitado de operadores. Perceba que faz todo sentido poder somar números inteiros, mas somar endereços? E multiplicar booleanos? A ideia é que nem todas as operações estão disponíveis para todos os tipos, como veremos a seguir.

Inteiros

São divididos em inteiros com sinal (int) e inteiros sem sinal (unsigned int, ou uint). Cada número pode ocupar até 256 bits de memória. Um ponto interessante, é que em Solidity é permitido definir o tamanho de uma variável deste tipo, variando de int8 a int256 (no caso do inteiro com sinal) e do unit8 ao unit256 (no caso do inteiro sem sinal).

A seguir, mostramos um exemplo de código utilizando números inteiros. Aqui, vale a pena esmiuçar a primeira linha para aprofundar o entendimento.

Operadores de números inteiros

Os números inteiros admitem um conjunto de operadores muito comuns a basicamente todas as linguagens de programação modernas, como segue:

Tipos de Operadores
OPERADORES ARITMÉTICOS +, -, unary - (apenas para inteiros com sinal), *, /, % (módulo), ** (exponenciação)
OPERADORES DE COMPARAÇÃO <=, <, ==, !=, >=, > (cujo resultado é um booleano, verdadeiro ou falso)
OPERADORES DE BIT &, |, ^ (ou exclusivo bit-a-bit), ~ (negação bit-a-bit :)
OPERADORES DE SHIFT << (shift à esquerda), >> (shift à direita)
VOCÊ SABIA? As operações bit-a-bit possuem diversas aplicações voltadas para o alto desempenho de uma aplicação, mas seu uso é incomum. Além disso, a operação de shift também é aplicada bit a bit e um uso muito comum é a multiplicação e divisão de um número. Ou seja, quando um shift à esquerda é aplicado a um número inteiro (em binário) ele é multiplicado por dois. Da mesma forma, quando um shift à direita é aplicado, o número inteiro é dividido por dois, mantendo-se apenas a parte inteira do resultado.

VOCÊ SABIA? A quantidade de bits de memória define o intervalo de números aceitos naquele tipo. Por exemplo, em um int8 pode-se armazenar números que vão apenas de -127 a 128, enquanto em um int32 é possível armazenar número de -2.147.483.648 a 2.147.483.647. Naturalmente, se um programa precisar armazenar números maiores ou menores que os citados, deve-se utilizar tipos maiores. De forma complementar, se o programa armazena apenas valores pequenos, sugere-se o uso de tipos menores, o que ajudará a economizar memória e banda de internet no tráfego da informação pela rede.

Número de ponto fixo

São utilizados para números com representação decimal, os números fracionários. Dividem em dois tipos, os de ponto fixo com sinal (fixed), e os ponto fixo sem sinal (ufixed). Até a versão 0.8.17 do compilador Solidity, a mais atual no momento da criação deste curso, os números de ponto fixo ainda não são totalmente suportados pela linguagem. Eles podem ser declarados, mas não podem ser atribuídos.

Booleanos (bool)

O tipo booleano existe para permitir a construção de verificações lógicas, cujo resultado precisa ser descrito como verdadeiro (true) ou falso (false).

Seu uso é bastante amplo, e se aplica especialmente para fins de construções lógicas que envolvam comparação, a exemplo de quando um indivíduo digita sua senha, e essa precisa ser comparada com aquela armazenada no banco de dados. A resposta esperada só pode ser um booleano: se a senha está correta (true), ou errada (false).

Operando booleanos

Dado que os tipos booleanos representam apenas valores restritos a verdadeiro e falso (true e false), seus operadores são chamados de lógicos, e são os mesmos utilizados em várias linguagens como C, C++ e Java:

  1. ! (negação lógica)
  2. && (conjunção lógica, “e”)
  3. || (disjunção lógica, “ou”)
  4. == (igualdade)
  5. != (desigualdade)
VOCÊ SABIA? Nas conjunções (&&) e disjunções (||) aplica-se uma importante regra chamada de curto-circuito. Isso significa que quando há duas ou mais expressões sendo avaliadas e uma delas se mostra suficiente para uma conclusão, as demais não serão avaliadas.

Um exemplo bem simples é pensar que a aprovação de um aluno depende de média maior que 7, ou nota na prova final maior que 5.Escrevendo em código teríamos algo como:bool aprovado = media > 7 || final > 5.

Suponha que o aluno teve média = 8.Pela regra do curto-circuito, ao validar o primeiro lado da expressão, o outro lado não é verificado, pois já é possível concluir que o aluno está aprovado.

Endereços (address)

Este é um tipo muito incomum para outras linguagens de programação. No Solidity, ele existe por estar no contexto de contratos inteligentes. Uma variável do tipo address sempre deverá representar um endereço válido da rede Ethereum, a exemplo de 0xb794f5ea0ba39494ce839613fffba74279579268.

Perceba que se trata de um texto longo, no entanto, a aplicação de um tipo específico para endereços tem um propósito de torná-lo mais sofisticado que apenas um texto. Através deste tipo é permitido fazer operações, como transferências e verificação de saldo, como será mostrado em exemplos, adiante.

Operando Address (endereços)

Diferente dos tipos anteriores, o tipo address é um pouco mais complexo, pois visa permitir operações financeiras, essas operações são executadas através de funções exclusivas desse tipo, como discutiremos a seguir.

Para começar, vamos analisar melhor o tipo Address, que tem duas formas:

  1. address e
  2. address payable (endereço a pagar, em tradução direta).

Ambos representam um endereço da rede Ethereum com 20 bytes de tamanho, mas a diferença existe porque somente o payable pode efetivamente receber Ether, enquanto o primeiro não pode receber, apenas enviar.

No código, se você precisar de uma variável do tipo endereço e planeja enviar Ether para ela, declare seu tipo como address payable, como você poderá observar no exemplo a seguir, mais à frente, entenderemos detalhes do uso desse tipo.

Send

Send é a contrapartida de baixo nível da transfer. Se a execução falhar, o contrato atual não será interrompido com uma exceção, mas send retornará false.

AVISO Existem alguns perigos no uso de send: A transferência falha se a profundidade da pilha de chamadas estiver em 1024 (isso sempre pode ser forçado pelo chamador) e também falha se o destinatário ficar sem gás. Então, para fazer transferências Ether seguras, verifique sempre o valor de retorno do envio, use transferência ou ainda melhor: use um padrão onde o destinatário saca o dinheiro.

Array de bytes de tamanho fixo

Esse tipo trata de arrays de tamanho fixo até 32 bytes. Ele deve ser declarado como bytesx, onde x pode variar de 1 a 32 e representa o tamanho de bytes definidos para o vetor. Existem outros tipos de array em Solidity, mas são considerados tipos por referência, sendo tipo por valor apenas os de bytes.

Um membro definido para os arrays é o .length que retorna o comprimento da cadeia.

Operando array de bytes

Para os arrays de bytes são possíveis usar as operações:

  1. + : concatenação;
  2. == : comparação de igualdade;
  3. != : comparação de diferença.
  4. =: atribuição

Literais

Literais são valores fixos declarados nos contratos inteligentes. Existem diferentes tipos de literais para representar diferentes tipos de dados, como números inteiros, string de caracteres e booleanos. Veja exemplos:

  1. Números inteiros: 42, 0x2a (representação de hexadecimal de 42)
  2. String de caracteres: “Olá, mundo!”
  3. Booleano: true, false.

Além disso, existem alguns literais especiais em Solidity que são usados para representar valores especiais, como o endereço de um contrato inteligente ou o tamanho de um array.

Tipos por referência

Os tipos por referência são aqueles cujo valor é passado entre variáveis através da sua localização de armazenamento. Em vez de criar uma cópia do elemento original, como ocorre com os tipos de valor, os tipos por referência compartilham a mesma localização de armazenamento. Isso significa que as variáveis passam a apontar para o mesmo elemento e, portanto, quaisquer alterações feitas em uma dessas variáveis são instantaneamente refletidas em todos os outros pontos do código que apontam para a mesma referência.

Data location

Em Solidity, é necessário deixar explícito o local de compartilhamento da localização.. As opções de localização são:

  1. Em memória (memory) - usado para armazenar temporariamente os valores passados entre funções. O tempo de vida dos valores é limitado ao período em que a função está sendo utilizada, sendo descartados logo ao término da execução. Este tipo de dado, geralmente, é utilizado para passar valores para funções e só pode ser declarado dentro do escopo das funções. Não é adequado para armazenar valores permanentemente.

  2. Em armazenamento (storage) - utilizada quando o tempo de vida dos elementos é limitado ao tempo de vida do contrato, estando acessível enquanto o contrato estiver em execução. Os valores armazenados no 'storage' são visíveis para todos os contratos que interagem com o contrato atual e são armazenados na blockchain do Ethereum. É uma localização mais cara do que memory, quando consideramos as taxas de Gas, justamente por possuir um tempo maior de vida, o que também deve ser considerado no desenvolvimento.

MEMORY STORAGE
Local de armazenamento Na memória do contrato Na blockchain Ethereum
Duração do armazenamento Até o final da função que os usa O tempo de execução do contrato na blockchain.
Quem tem acesso Apenas a função que os usa Todos os contratos que interagem com o contrato
Custo de gás Menor Maior

Existem também a localização calldata, similar a localização memory, com a diferença que o dado não pode ser alterado durante a execução.

String

Trata-se do tipo que permite a escrita de textos de propósito geral, desde que contenha caracteres válidos; e são sempre escritos entre aspas. É um dos tipos mais comuns, e presente em todas as linguagens de programação modernas. Ele é um tipo de dados por referência e funciona de forma dinâmica, o que significa que o tamanho da string não é fixo e pode variar conforme a necessidade.

No Solidity, sua escrita aceita aspas simples ou duplas. As strings podem representar elementos concretos como o nome de uma pessoa: “Manoela Silva”; uma placa de veículo: 'PGG-3A57'; um CNPJ: “12.123.595/0001-34”, ou algo que não tenha um sentido direto, como: “_=¨`847lkajfa;:k6f+11av9jaqy[]á67945”.

Em resumo, strings, como seu nome já diz, representam cadeiras (ou sequências) de caracteres para fins quaisquer.

Tipos de mapeamento (mapping)

O mapping é um tipo de dado que permite você criar uma estrutura de dados capaz de mapear uma chave para um valor. É uma estrutura de dados bastante útil pois permite que você armazene e acesse informações de forma rápida e eficiente.

Uma analogia para entender o funcionamento do mapping é pensar nele como um índice de um livro. No índice de um livro, cada página é acessada através de um tópico (que é a chave do índice).

Assim como no índice de um livro, o mapping permite armazenar um conjunto de valores de um determinado tipo (por exemplo, páginas de um livro) e acessá-los através de uma chave de outro tipo (por exemplo, tópicos).

Um mapping é declarado da seguinte maneira:

mapping(tipoDaChave => tipoDoValor) nomeDoMapping

Onde:

  1. tipoDaChave é o tipo de dado da chave. Ela pode ser de qualquer tipo, exceto arrays, structs e outros mappings.
  2. tipoDoValor é o tipo de dado do valor cujo os valores não possuem restrições quanto ao tipo.
  3. nomeMapping é o nome da variável do mapping.

Estruturas de controle

Você sabia que as estruturas de controle são super importantes em qualquer linguagem de programação? Elas permitem que os desenvolvedores criem algoritmos que tomem decisões e executem ações de acordo com as condições específicas. A linguagem Solidity, usada para escrever contratos inteligentes no Ethereum, tem várias estruturas de controle que permitem criar lógicas complexas para os contratos.

Uma das estruturas de controle mais comuns é o "if-else", uma estrutura condicional, que executa uma ação específica se a condição for verdadeira e outra ação se a condição for falsa. Por exemplo:

Outra estrutura de controle comum é o "loop" de repetição, que faz uma sequência de ações repetir até que uma condição seja satisfeita. Solidity tem três tipos de loop:

  1. for: comando de repetição que permite executar uma série de instruções várias vezes. Usado, geralmente, quando se conhece o número de vezes que será repetido ou para iterar um array.

  2. while: comando de repetição que permite executar uma série de instruções enquanto uma determinada condição for verdadeira. A condição é avaliada no início de cada iteração do laço.

  3. do-while: funciona de forma muito semelhante ao 'while', porém a condição é avaliada ao final da iteração do laço, garantindo que a série de instruções é executada ao menos uma vez.
