Sintaxe

Tipos de dados

Na linguagem de programação Solidity, o programador precisa especificar o tipo de cada variável que será usada, por isso ela é dita uma linguagem estaticamente tipada. Exemplos de outras linguagens tipadas incluem o Java, C, C++, dentre várias outras.

Dentre as principais vantagens de se trabalhar uma linguagem estaticamente tipada estão a facilidade de identificar erros no código e maior facilidade na leitura e entendimento do código em si.

O Solidity trabalho com três tipos principais de dados:

Tipos de valor

Os Value Types são utilizados para definir o tipo básicos de variáveis. Representam os tipos mais comuns, existentes em basicamente todas as linguagens de programação modernas. A seguir, será explorado alguns tipos e você entenderá como ele é representado em detalhes, e seu propósito de uso. Para ter acesso a detalhes de todos os tipos, acesse a documentação oficial do Solidity: https://docs.soliditylang.org/en/v0.8.17/types.html

Além disso, cada tipo admite um conjunto bem limitado de operadores. Perceba que faz todo sentido poder somar números inteiros, mas somar endereços? E multiplicar booleanos? A ideia é que nem todas as operações estão disponíveis para todos os tipos, como veremos a seguir.

Inteiros

São divididos em inteiros com sinal (int) e inteiros sem sinal (unsigned int, ou uint). Cada número pode ocupar até 256 bits de memória. Um ponto interessante, é que em Solidity é permitido definir o tamanho de uma variável deste tipo, variando de int8 a int256 (no caso do inteiro com sinal) e do unit8 ao unit256 (no caso do inteiro sem sinal).

A seguir, mostramos um exemplo de código utilizando números inteiros. Aqui, vale a pena esmiuçar a primeira linha para aprofundar o entendimento.

Número de ponto fixo

São utilizados para números com representação decimal, os números fracionários. Dividem em dois tipos, os de ponto fixo com sinal (fixed), e os ponto fixo sem sinal (ufixed). Até a versão 0.8.17 do compilador Solidity, a mais atual no momento da criação deste curso, os números de ponto fixo ainda não são totalmente suportados pela linguagem. Eles podem ser declarados, mas não podem ser atribuídos.

Booleanos (bool)

O tipo booleano existe para permitir a construção de verificações lógicas, cujo resultado precisa ser descrito como verdadeiro (true) ou falso (false).

Seu uso é bastante amplo, e se aplica especialmente para fins de construções lógicas que envolvam comparação, a exemplo de quando um indivíduo digita sua senha, e essa precisa ser comparada com aquela armazenada no banco de dados. A resposta esperada só pode ser um booleano: se a senha está correta (true), ou errada (false).

Endereços (address)

Este é um tipo muito incomum para outras linguagens de programação. No Solidity, ele existe por estar no contexto de contratos inteligentes. Uma variável do tipo address sempre deverá representar um endereço válido da rede Ethereum, a exemplo de 0xb794f5ea0ba39494ce839613fffba74279579268.

Perceba que se trata de um texto longo, no entanto, a aplicação de um tipo específico para endereços tem um propósito de torná-lo mais sofisticado que apenas um texto. Através deste tipo é permitido fazer operações, como transferências e verificação de saldo, como será mostrado em exemplos, adiante.

Send

Send é a contrapartida de baixo nível da transfer. Se a execução falhar, o contrato atual não será interrompido com uma exceção, mas send retornará false.

Array de bytes de tamanho fixo

Esse tipo trata de arrays de tamanho fixo até 32 bytes. Ele deve ser declarado como bytesx, onde x pode variar de 1 a 32 e representa o tamanho de bytes definidos para o vetor. Existem outros tipos de array em Solidity, mas são considerados tipos por referência, sendo tipo por valor apenas os de bytes.

Um membro definido para os arrays é o .length que retorna o comprimento da cadeia.

Tipos por referência

Os tipos por referência são aqueles cujo valor é passado entre variáveis através da sua localização de armazenamento. Em vez de criar uma cópia do elemento original, como ocorre com os tipos de valor, os tipos por referência compartilham a mesma localização de armazenamento. Isso significa que as variáveis passam a apontar para o mesmo elemento e, portanto, quaisquer alterações feitas em uma dessas variáveis são instantaneamente refletidas em todos os outros pontos do código que apontam para a mesma referência.

String

Trata-se do tipo que permite a escrita de textos de propósito geral, desde que contenha caracteres válidos; e são sempre escritos entre aspas. É um dos tipos mais comuns, e presente em todas as linguagens de programação modernas. Ele é um tipo de dados por referência e funciona de forma dinâmica, o que significa que o tamanho da string não é fixo e pode variar conforme a necessidade.

No Solidity, sua escrita aceita aspas simples ou duplas. As strings podem representar elementos concretos como o nome de uma pessoa: “Manoela Silva”; uma placa de veículo: 'PGG-3A57'; um CNPJ: “12.123.595/0001-34”, ou algo que não tenha um sentido direto, como: “_=¨`847lkajfa;:k6f+11av9jaqy[]á67945”.

Em resumo, strings, como seu nome já diz, representam cadeiras (ou sequências) de caracteres para fins quaisquer.

Tipos de mapeamento (mapping)

O mapping é um tipo de dado que permite você criar uma estrutura de dados capaz de mapear uma chave para um valor. É uma estrutura de dados bastante útil pois permite que você armazene e acesse informações de forma rápida e eficiente.

Uma analogia para entender o funcionamento do mapping é pensar nele como um índice de um livro. No índice de um livro, cada página é acessada através de um tópico (que é a chave do índice).

Assim como no índice de um livro, o mapping permite armazenar um conjunto de valores de um determinado tipo (por exemplo, páginas de um livro) e acessá-los através de uma chave de outro tipo (por exemplo, tópicos).

Um mapping é declarado da seguinte maneira:

mapping(tipoDaChave => tipoDoValor) nomeDoMapping
