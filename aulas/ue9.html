Comportamento da função

As funções podem apresentar diferentes comportamentos em relação ao estado da rede. O estado são os dados armazenados na memória da Blockchain. Considerando as funções, alguns parâmetros podem ser utilizados para definição de como acessar/modificar o estado.

Se nenhuma palavra reservada for utilizada, a função é executada no modo transaction, ou seja, ela pode retornar, gravar ou alterar dados. A execução das funções transaction sempre envolvem pagamento de gás, mesmo que seja apenas uma leitura de valores.

Existem as funções do tipo view que são aquelas incapazes de modificar o estado do elemento a que se referem. Elas são executadas no modo call, isso faz com que não tenha custo de gás envolvido.

Outro tipo utilizado, são as funções definidas como{" "} pure. Elas são as mais restritivas e seu uso não permite ler ou alterar dados. Sua execução também ocorre no modo call e não envolve custo de gás.

As funções pagáveis (payable) permitem receber fundos, ou seja, recebem Ether (Eth) e/ou outros tokens.

Funções e variáveis intrínsecas (Build-in)

As variáveis/funções intrínsecas (build-in) são implementações oferecidas pela própria linguagem de programação. No Solidity, você deve usá-las quando precisar que dados sejam tratados como dados de um bloco ou de uma transação.

Construtores

O construtor é um tipo especial de função que é executada apenas uma vez quando o contrato é implantado e tem como objetivo inicializar as variáveis de estado. Cada contrato possui um único construtor.

Para sua definição é usada a palavra reservada constructor. Caso nenhum construtor seja definido, Solidity considera um construtor vazio como padrão. Cada contrato possui um único construtor.

Um exemplo bastante utilizado é o modificador que garante que uma função só será executada se for chamada pelo dono do contrato.
