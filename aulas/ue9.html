Sintaxe

Solidity é uma linguagem de programação orientada a objetos com muitos recursos e está em constante evolução. Você vai conhecer um pouco mais sobre a sintaxe de funções, construtores e modificadores. É importante ficar atento às mudanças, pois as evoluções da linguagem são muito rápidas. Mantenha-se atualizado consultando a documentação em: https://solidity-portuguese.readthedocs.io/pt/latest/

Funções

No Solidity, assim como em outras linguagens de programação, as funções representam possíveis ações de um contrato inteligente. Na prática, elas são usadas para agrupar trechos de código que podem ser chamados quando necessário.

Analisando a imagem, pode-se destacar:

  1. Uso da palavra reservada function para definição da função.
  2. As funções recebem um nome de identificação para serem chamadas
  3. Entre parênteses, podem ter parâmetros de entrada que são definidos com tipo e nome.
  4. Elas podem retornar um ou mais valores. Nesse caso, usa-se a palavra reservada returns e os tipos dos valores retornados devem ser especificados entre parênteses.
  5. Há funções que não retornam nenhum valor.

Outras duas propriedades relacionadas às funções são a visibilidade e o identificador de estado. A princípio, é necessário saber que a visibilidade estabelece onde a função poderá ser acessada e o identificador de estado define como a função tem acesso aos dados. Por padrão a visibilidade das funções é public, então se você não especificar na função a visibilidade, ela estará visível dentro e fora do contrato.

Comportamento da função

As funções podem apresentar diferentes comportamentos em relação ao estado da rede. O estado são os dados armazenados na memória da Blockchain. Considerando as funções, alguns parâmetros podem ser utilizados para definição de como acessar/modificar o estado.

Se nenhuma palavra reservada for utilizada, a função é executada no modo transaction, ou seja, ela pode retornar, gravar ou alterar dados. A execução das funções transaction sempre envolvem pagamento de gás, mesmo que seja apenas uma leitura de valores.

Existem as funções do tipo view que são aquelas incapazes de modificar o estado do elemento a que se referem. Elas são executadas no modo call, isso faz com que não tenha custo de gás envolvido.

Outro tipo utilizado, são as funções definidas como pure. Elas são as mais restritivas e seu uso não permite ler ou alterar dados. Sua execução também ocorre no modo call e não envolve custo de gás.

Você pode estar se perguntando se existe alguma função especial para ações que envolvem transações financeiras? A resposta é sim!

As funções pagáveis (payable) permitem receber fundos, ou seja, recebem Ether (Eth) e/ou outros tokens.

Funções e variáveis intrínsecas (Build-in)

As variáveis/funções intrínsecas (build-in) são implementações oferecidas pela própria linguagem de programação. No Solidity, você deve usá-las quando precisar que dados sejam tratados como dados de um bloco ou de uma transação.

Essas variáveis são acessíveis através de instruções e você conhecerá algumas a seguir.

  1. block.blockhash(uint blockNumber): Essa função retorna o hash do bloco fornecido (funciona para os últimos 256 blocos).

  2. block.chainid (uint): Tem como função retornar um 'id' (identificador), do tipo uint, da Blockchain atual. Cada Blockchain possui um número único para identificá-la, assim você pode, facilmente, saber qual rede está utilizando. Por exemplo: A Blockchain Ethereum Mainnet tem o id igual a 1, já a Binance Smart Chain (BSC) possui id igual a 56.

Dica Você pode consultar os Chain ids em: https://chainlist.org/
Também é possível inserir sua rede e receber seu id.
  1. block.coinbase(address): Após realizar deploy, seu contrato será incluído na rede por um minerador. Essa função retorna o endereço do minerador responsável, tornando possível alguma interação com ele.

  2. block.difficulty(uint): Responsável por retornar o grau de dificuldade do bloco, O parâmetro de dificuldade de mineração é utilizado para regulação de emissão de moedas e é constantemente atualizado.

  3. block.gaslimit: Retorna a quantidade de gás disponível para o bloco atual. Assim, é possível saber se o bloco atual conseguirá processar seu contrato ou não.

  4. block.number(uint): Retorna o número do bloco atual.

Dica Acesse: https://etherscan.io/blocks e confira quantos blocos existem na rede Ethereum atualmente.
  1. block.timestamp(uint): Retorna o timestamp em segundos do bloco atual.
  2. msg.sender(adress): Retorna o endereço da mensagem atual.
  3. msg.value(uint): Retorna o número de Wei (unidade do eth) enviados na transação.
  4. tx.origin(address): Retorna o remetente da transação. Parecido com msg.sender(adress):com a diferença que tx.origin só refere-se à endereço de carteiras.
  5. tx.gasprice(unit): Retorna o valor do gás para a transação corrente.
  6. address(this).balance: Retorna o saldo do contrato corrente.

Construtores

O construtor é um tipo especial de função que é executada apenas uma vez quando o contrato é implantado e tem como objetivo inicializar as variáveis de estado. Cada contrato possui um único construtor.

Para sua definição é usada a palavra reservada constructor. Caso nenhum construtor seja definido, Solidity considera um construtor vazio como padrão. Cada contrato possui um único construtor.

Um exemplo bastante utilizado é o modificador que garante que uma função só será executada se for chamada pelo dono do contrato.
